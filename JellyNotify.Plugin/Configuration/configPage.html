General

JellyNotify notifies you in real time about what's happening with your content requests: when they're created, approved or declined, how the download is going, and when it's finally available — right inside Jellyfin, and optionally on Telegram, Discord, or WhatsApp too.

Connected services

To get notified about your requests (created, approved, declined, available...), connect Seerr. To know download progress, connect Sonarr (series) and Radarr (movies). If you also want these alerts to reach you outside the server, connect a Telegram, Discord, or WhatsApp bot from the notification bell's settings panel.

Serr

⚡Seerr notifies JellyNotify instantly via webhook — there's no polling interval to set. A single check runs once when Jellyfin starts, only to catch up on anything that changed while it was off.

Discord

How to set this up (step by step)
  1. Go to the Discord Developer Portal and click New Application.
  2. Open the Bot tab, click Reset Token, and copy it into Bot token below.
  3. Open the OAuth2 tab and copy the Client ID and Client Secret into the matching fields below.
  4. Still on the OAuth2 tab, under Redirects, add the exact URL shown in OAuth redirect URI below, then save it there too.
  5. Save the configuration here first, then click Invite bot to my server and add the bot to a Discord server where your Jellyfin users already are — Discord requires this before the bot can DM anyone.
  6. Users can now click Connect Discord from the notification bell's settings panel.
⚠️Created from Server Settings → Integrations → Webhooks in Discord — not the Developer Portal, which is for the next step. Example: https://discord.com/api/webhooks/123456789012345678/AbCdEfGh...
💬Bot tab of the Developer Portal → Reset Token. It's a long string — copy it exactly, it's only shown once.
💬OAuth2 tab of the Developer Portal, just a number. Example: 123456789012345678
💬Right next to the Client ID, same OAuth2 tab. Treat it like a password.
💬Auto-filled from the address you're using right now to view this page — it does not need to be public or use HTTPS, since your own browser follows this link, Discord's servers never call it directly. If people also connect from a different address (a public domain, for example, while you're on a local IP), add that address's callback URL here too, as a separate entry in Discord's Redirects list — it must match exactly whichever one was used, or Discord will reject it.
The webhook above is a global/admin-only broadcast. The bot token + Client ID/Secret together unlock "Connect Discord" in every user's notification panel — the bot must share a server with each user it DMs (Discord's own anti-spam rule).

Telegram

How to set this up (step by step)
  1. Open Telegram and start a chat with @BotFather.
  2. Send /newbot and follow the prompts to choose a name and a username (must end in "bot").
  3. BotFather replies with a token — copy it into Bot token below.
  4. Copy the bot's @username (without the @) into Bot username below.
  5. Save the configuration. Users can now click Connect Telegram from the notification bell's settings panel — it opens a chat with your bot; tapping Start finishes the connection automatically.
💬@BotFather gives you this right after creating the bot with /newbot. Example: 123456789:AAExampleTokenXXXXXXXXXXXXXXXXXXXXX
💬The username you gave the bot in BotFather, without the @ symbol. Example: MyServerBot
Open a chat with your bot in Telegram, send it any message, wait a few seconds, then click Detect automatically. If nothing is found, send another message to the bot first and try again.
💬Only needed for the admin's own test button — each user connects their personal account from the bell without needing this. Group chat IDs start with -100.
Users connect from the notification panel with one click — it opens a chat with your bot, and tapping Start finishes the connection automatically (no ID to type).

WhatsApp

How to set this up (step by step)
  1. Create a free account at developers.facebook.com (Meta for Developers).
  2. Click My Apps → Create App, choose the Business type, and add the WhatsApp product when prompted.
  3. Open WhatsApp → API Setup. Meta gives you a free test number here — copy its Phone Number ID and the temporary Access Token into the fields below (a permanent token needs a System User, see Meta's docs; the temporary one works fine to get started but expires after 24h).
  4. Also copy the test number itself (E.164, no leading +) into Admin/bot WhatsApp number below.
  5. Make up any random text and put it in Verify Token below — it's just a shared password between JellyNotify and Meta, not sensitive.
  6. Save the configuration here first, then go to WhatsApp → Configuration in Meta, click Edit next to Webhook, and paste in the Webhook URL shown below plus the same Verify Token you just chose.
  7. Still in that Meta screen, click Manage next to "Webhook fields" and subscribe to messages.
  8. On Meta's free tier you can only message up to 5 phone numbers, added manually under API Setup → "To" — add your own number there while testing. For a real server with many users, you'll need to verify your business and get a production number.
💬The number Meta gives you (or your production number later), in international format but without the + symbol. Example: 34612345678
💬Copied from WhatsApp → API Setup in Meta. The one shown there by default is temporary (expires in 24h); a permanent one needs a System User.
⚠️This is not the phone number — it's a separate numeric ID on that same Meta page. Example: 123456789012345
💬You make this up yourself — it just needs to match whatever you enter in Meta's webhook config. Example: jellynotify-verify-2026
💬Already filled in by the plugin — nothing to type here. Save this configuration first, then copy it into Meta along with the same Verify Token above.
⚠️Meta needs to be able to reach this address from the internet, over HTTPS with a valid certificate (a self-signed one won't work). If your Jellyfin is only reachable on your local network (an address like 192.168.x.x or similar) or has no real certificate, this URL won't work and subscribing to the webhook will fail in Meta. You'll need to expose the server to the internet with a real domain and certificate (port forwarding, a reverse proxy, or a tunnel like Cloudflare Tunnel) before this step can be completed.
Without the Cloud API fields above, WhatsApp is link-only: users can message your number, but the bot can't reply automatically (no welcome message, no real notifications). Fill them in to make WhatsApp a fully working channel.
